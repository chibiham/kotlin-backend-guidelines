# Kotlin Spring Boot 開発指針

本ドキュメントは、Kotlin + Spring Boot によるバックエンド開発における設計方針・コーディング規約をまとめたものです。

## 目的

- チーム内での設計・実装方針の統一
- 新規メンバーのオンボーディング支援
- レビュー時の判断基準の明確化

## 対象読者

- 本プロジェクトに参加する開発者
- Java の経験があり、Kotlin での開発を始める方

## 読み方

**初めて読む場合**: 02 → 03 → 04 の順で読むことを推奨します。Kotlin の書き方と依存性注入の方針を理解した上で、他のドキュメントに進んでください。

**リファレンスとして使う場合**: 各ドキュメントは独立して参照できるように書かれています。必要な箇所を直接参照してください。

## 原則

本ドキュメント全体を通じて、以下の原則を重視しています。

1. **テスタビリティ**: テストしやすい設計を優先する
2. **明示性**: 暗黙的な動作より明示的な記述を好む
3. **標準準拠**: RFC や業界標準に従い、車輪の再発明を避ける
4. **シンプルさ**: 必要十分な複雑さに留め、過度な抽象化を避ける
